# Ask vs Act

Default to action. Only ask when the answer genuinely changes what you build.

**Ask when:** scope is genuinely ambiguous, architecture has real tradeoffs, intent is unclear, action is destructive/irreversible.

**Act when:** the task is clear, the next step is obvious, you just need to do the thing.

Before asking: "If I just did the obvious thing, would the user be annoyed or relieved?" If relieved — act.
